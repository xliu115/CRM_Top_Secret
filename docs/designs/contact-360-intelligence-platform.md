## Overview

Contact 360 is a comprehensive, AI-generated intelligence dossier for any contact in the CRM. It combines CRM data (interactions, meetings, signals, nudges, sequences, firm relationships) with live web intelligence (background, news, job changes) to produce strategic briefings that help Partners prepare for meetings and make informed outreach decisions.

The feature is delivered through three interfaces:

1. **Full Contact 360** вҖ” 7-section dossier on the contact detail page
2. **Quick 360** вҖ” Insight summary + talking points in the Ask Anything chat
3. **Mini 360** вҖ” 3-section snippet in morning briefing emails

## Feature Set

### Contact 360 (Full Dossier)

7 collapsible sections generated by `generateContact360()`:


| Section                        | ID               | Content                                                          |
| ------------------------------ | ---------------- | ---------------------------------------------------------------- |
| Person Profile                 | `profile`        | Name, title, company, importance, web background                 |
| Relationship Overview          | `relationship`   | Interaction count, first/last contact dates, relationship health |
| Communication Timeline         | `timeline`       | Chronological interaction history with sentiment                 |
| Firm-Wide Connections          | `firm`           | Other Partners who know this contact, intensity levels           |
| News and Signals               | `signals`        | CRM signals + live web news                                      |
| Open Threads & Recommendations | `actions`        | Active nudges, sequences, upcoming meetings, next steps          |
| Talking Points                 | `talking_points` | 3 specific conversation starters for meetings                    |


**Access:** Contact detail page вҶ’ "Contact 360" button in profile header

**Additional capabilities on the dossier card:**

- Inline follow-up chat (passes full 360 context)
- Context-aware email draft (pre-populated with relationship context)
- Share dossier via email (formatted HTML via Resend)

### Company 360

5 collapsible sections generated by `generateCompany360()`:


| Section                    | ID                | Content                                                     |
| -------------------------- | ----------------- | ----------------------------------------------------------- |
| Company Overview           | `overview`        | Industry, size, website, tracked contacts                   |
| Firm Coverage Map          | `coverage`        | Which Partners cover which contacts                         |
| Relationship Health Matrix | `health`          | Interaction frequency, cold contacts, at-risk relationships |
| Signals and News           | `signals`         | Company-level signals + web news                            |
| Strategic Recommendations  | `recommendations` | Coverage gaps, coordination opportunities                   |


**Access:** Company detail page вҶ’ "Company 360" button in company header

### Quick 360 (Chat)

Delivered in the Ask Anything conversation thread via `generateQuick360()`:

- **Insight Summary** вҖ” 3-5 sentence strategic paragraph
- **Talking Points** вҖ” 3 conversation starters
- **Quick Action buttons** вҖ” Full Contact 360, Company 360, Draft Email, Share Dossier

**Access:** Dashboard morning briefing вҶ’ "Quick 360 вҖ” {name}" button вҶ’ navigates to chat

### Mini 360 (Morning Briefing)

3-section snippet generated by `generateMini360()`, embedded in daily briefing emails:


| Section          | Content                           |
| ---------------- | --------------------------------- |
| Person Profile   | Name, title, company, importance  |
| Firm Connections | Other Partners with relationships |
| Key Signals      | Latest signals and news           |


**Access:** Automatically included for top 3 action contacts in morning briefing emails

Original Contact 360 plan (6 sections, button + chat intent) PLUS 6 accepted expansions:

1. **Morning Briefing Integration** -- mini-360 snippets for top contacts in daily email
2. **Talking Points section** -- 3 conversation starters as section 7
3. **Ask Follow-up inline** -- chat input at bottom of dossier with pre-loaded 360 context
4. **Context-aware Email Draft** -- "Draft Email" button on dossier that includes 360 context
5. **Company 360** -- company-level aggregated dossier on company detail page
6. **Share Dossier via Email** -- send formatted dossier to colleagues via Resend

Two deferred:

- Pre-Meeting Auto-Push with 360 context (next sprint, depends on cron calendar integration)
- Relationship Risk Score (next sprint, feeds into Connection Strength Scoring TODO)

## Intent Detection (Chat)

The chat route detects 360 queries via regex:

```
Contact 360:  /tell me everything about|contact ?360|recap|dossier|
               what do you know about|full intel on|brief me on|quick ?360/i

Company 360:  /company ?360|company dossier|tell me about the company|
               company intelligence/i
```

Name extraction strips the intent keywords and "for" to isolate the contact/company name, then matches via `contactRepo.search()`.

## Quick Actions in Chat

Quick Actions are embedded as a structured HTML comment in the chat response:

```html
<!--QUICK_ACTIONS:[{"label":"Full Contact 360","href":"/contacts/abc"},...]-->
```

The `AssistantReply` component extracts this marker, strips it from the markdown content, and renders the actions as styled pill buttons matching the dashboard briefing design.

## Template Fallbacks

When the LLM is unavailable (quota exceeded, API error), each function falls back to a template-generated response using the same CRM data:

- `generateContact360Template()` вҖ” Assembles 7 sections from raw data
- `generateCompany360Template()` вҖ” Assembles 5 sections from aggregated data
- `generateMini360Template()` вҖ” Assembles 3 sections for briefing
- `generateQuick360Template()` вҖ” Assembles insight + talking points

Fallbacks produce the same data structure as LLM responses, ensuring the UI always renders.

## Email Integration

### Morning Briefing (Mini 360)

In `briefing-service.ts`, for the top 3 action contacts:

1. Fetch interactions, signals, web news in parallel
2. Call `generateMini360()` to produce 3-section snippet
3. Render via `buildMini360Html()` as styled inline cards
4. Inject into briefing email under "Contact Intelligence" section

### Share Dossier

`POST /api/contacts/{id}/contact360/share` accepts:

- `recipientEmail` вҖ” destination email address
- `result` вҖ” full `Contact360Result` object

Renders the dossier as formatted HTML via `buildContact360EmailHtml()` and sends via Resend.

## LLM Prompt Design (Company 360, 5 sections)

```
You are a senior intelligence analyst preparing a company-level relationship
dossier for a consulting Partner preparing for a quarterly business review.

Sections:
1. Company Overview -- industry position, size, recent trajectory
2. Firm Coverage Map -- which partners know people here, depth of relationships
3. Relationship Health Matrix -- per-contact health summary, who's warm/cold
4. Signals and News -- aggregated news, job changes, market moves
5. Strategic Recommendations -- cross-sell gaps, relationship risks, opportunities

Output: same ---SECTIONS--- JSON format.
```
